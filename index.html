02 / A2V ML Platform

Account2Vec — turning fleet telemetry into a
reusable ML primitive.

Every ML effort in Azure Storage was rebuilding its own features from the same raw telemetry. Account2Vec is an unsupervised autoencoder that compresses ~100 behavioral metrics per account into a dense embedding — then extends hierarchically to subscriptions and customers (TPIDs), so similarity, anomaly, forecasting, and LLM-reasoning teams all build on one shared representation.

about

Hi, I'm Sarthak.

I work on Azure Storage analytics and ML systems — customer health, behavioral embeddings, service-level dashboards, revenue correctness, and operationalizing analytics into agents and executive-ready insights.

I'm drawn to problems where the data is ambiguous and the stakes are real — where the signal is buried in noise, the decision has to ship, and the result shows up on someone's dashboard or someone's customer call.

Nine years in, I've moved from CLTV models at Mu Sigma to planogram optimization at Walmart to Azure Storage reliability, foundational ML platforms, and GenAI at Microsoft. The thread: I like turning ambiguity into a system — and I care a lot about whether the system actually moves the number it claimed it would.

Right now, two things energize me most: production LLM agents — grounded, evaluable, safe enough for real enterprise users — and foundational ML platforms, the kind of representation work where one model becomes the primitive ten downstream teams build on.
