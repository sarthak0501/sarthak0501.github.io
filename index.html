// PROJECT 02   ·   Account2Vec — Foundational Embedding Platform for Azure Storage ML PLATFORM
01 · The Problem

~100 raw metrics per account. No unified way to reason about millions of them.

Every ML effort in Azure Storage — customer similarity, anomaly detection, forecasting, workload segmentation — was rebuilding its own feature set from the same raw telemetry. There was no shared representation of what an "account" actually looks like at scale.

02 · The Build

Hierarchical autoencoders turning fleet telemetry into a reusable ML primitive.

  • Unsupervised autoencoder compressing ~100 metrics → dense embedding
  • Hierarchical: Account2Vec → Subscription2Vec → Customer2Vec (TPID)
  • Owned the AccountFeatureMatrix pipeline (~40–50 behavioral features across transactions, capacity, auth, latency, config)
  • Fleet-scale training on millions of accounts via Synapse + PyTorch
  • Drift / coverage / staleness monitoring and alerting
  • Authored full design docs: motivation, modeling, scalability tradeoffs
03 · The Impact

A platform primitive — not a one-off model — powering the next generation of Storage ML.

M+Azure Storage accounts embedded at fleet scale
3tiers: account · subscription · customer (TPID)
5+downstream uses: similarity, anomaly, forecasting, LLM reasoning, MCC workload agents
★positioned as org-wide platform, with full drift + coverage monitoring
Tech Stack PyTorch Autoencoders Representation Learning Azure Synapse Azure Data Explorer / KQL Feature Engineering Embeddings Python Drift Monitoring Hierarchical Modeling

Hi, I'm Sarthak.

I work on Azure Storage analytics and ML systems — customer health, behavioral embeddings (Account2Vec), service-level dashboards, revenue correctness, and operationalizing analytics into agents and executive-ready insights.

I'm drawn to problems where the data is ambiguous and the stakes are real — where the signal is buried in noise, the decision has to ship, and the result shows up on someone's dashboard or someone's customer call.

Nine years in, I've moved from CLTV models at Mu Sigma to planogram optimization at Walmart to Azure Storage reliability, foundational ML platforms (Account2Vec), and GenAI at Microsoft. The thread: I like turning ambiguity into a system — and I care a lot about whether the system actually moves the number it claimed it would.

Right now I'm most energized by two things: production LLM agents (grounded, evaluable, safe enough for real enterprise users), and foundational ML platforms — the kind of representation work where one model becomes the primitive that ten downstream teams build on.
